## Summary

Three log/return sites that sliced raw model output (`raw[:300]`, `raw[:200]`, `raw.strip()`) have been replaced with `_output_fingerprint(raw)` returning `{"hash": sha256(raw)[:16], "length": len(raw)}`. Logs reference the hash for correlation; user-facing reasoning strings reference the same hash with operator-readable context.

## Defense-in-depth motivation

Three concrete failure modes the previous code enabled:

1. **Operator PII leakage** — model output can echo NETWORK CONTEXT, which contains hostnames, MACs, SSIDs, and operator-supplied notes. A parse-failure log preview would write those into `agent_actions.log` regardless of file permissions on the deployment.
2. **Prompt-injection payload echo** — a successful prompt-injection attempt against the model would cause the injected payload to appear in the model output. Logging the raw output would *persist* the payload to disk and surface it to any downstream UI that renders the reasoning string.
3. **Cross-tenant leakage** — in any future multi-operator deployment, a parse-failure log preview from operator A's network would land in a shared log stream readable by operator B.

The fingerprint preserves diagnostic value (operators correlating a user-facing reasoning string back to the structured error log) without the leakage surface.
